# Security Policy

## Supported Versions

As the project is currently in its early development phase, only the latest development branch is considered supported.

Once stable releases become available, this section will list supported versions and their security maintenance status.

| Version | Supported |
|----------|-----------|
| Development | ✅ |
| Stable releases | To be announced |

---

## Reporting a Vulnerability

The Open Manuscript Initiative (OMI) takes security seriously.

If you discover a potential security vulnerability, **please do not report it through public GitHub issues**.

Instead, contact the project maintainers privately with as much information as possible, including:

- a description of the vulnerability;
- steps to reproduce it;
- affected components;
- potential impact;
- proof-of-concept code (if available);
- suggested mitigation (optional).

We will acknowledge receipt of your report as soon as reasonably possible.

---

## Responsible Disclosure

We ask security researchers to:

- avoid publicly disclosing vulnerabilities until a fix is available;
- make a reasonable effort to provide sufficient information for reproduction;
- avoid accessing, modifying, or deleting data that does not belong to you;
- respect the privacy of users and contributors.

We are committed to working collaboratively with security researchers.

---

## Security Priorities

OMI aims to follow security best practices, including:

- secure authentication;
- least-privilege design;
- secure defaults;
- dependency management;
- encrypted communications;
- protection of personal information;
- secure handling of uploaded files;
- defense against common web vulnerabilities (OWASP Top 10).

---

## Third-Party Components

OMI relies on open standards and may integrate with third-party services.

Security issues affecting external software should also be reported to the respective maintainers when appropriate.

---

## Security Updates

Security fixes will be prioritized whenever practical.

Critical vulnerabilities may be fixed outside the normal development cycle.

---

## Scope

This policy applies to:

- the OMI core;
- official plugins;
- official integrations;
- project documentation where security implications exist.

Third-party plugins and unofficial extensions are outside the scope of this policy unless explicitly stated.

---

## Acknowledgements

We appreciate responsible security research and value the contributions of everyone helping make OMI more secure.
