# OMI Brand Guidelines

**Open Manuscript Initiative**  
Version 1.0  
Last updated: July 2026

---

# Purpose

These guidelines define the visual identity of the Open Manuscript Initiative (OMI).

Their purpose is to ensure a consistent appearance across all official communication, including:

- Website
- Documentation
- GitHub
- Social media
- Presentations
- Publications
- Printed materials

Every visual asset should follow these guidelines.

---

# Brand Philosophy

OMI represents a modern approach to scholarly communication.

The visual identity should communicate:

- openness
- clarity
- trust
- scientific quality
- accessibility
- interoperability
- long-term sustainability

The design language should remain simple, elegant, and timeless.

Avoid unnecessary decorative elements.

---

# Logo

## Primary Logo

The primary logo combines:

- the OM symbol
- the Open Manuscript Initiative wordmark
- the project tagline (optional)

This is the preferred logo for most applications.

File:

```
logo/omi-logo-master.svg
```

---

## Dark Version

Use on light backgrounds.

```
logo/omi-logo-dark.svg
```

---

## Light Version

Use on dark backgrounds.

```
logo/omi-logo-light.svg
```

---

## Symbol

The symbol may be used independently when space is limited.

Examples:

- favicon
- application icon
- GitHub avatar
- social profile image

```
logo/omi-symbol.svg
```

---

## Monochrome

Use only when full-color reproduction is not possible.

---

# Clear Space

Always leave free space around the logo.

Minimum clear space equals the height of the green leaf.

No text, graphics or borders may enter this area.

---

# Minimum Size

Digital

| Asset | Minimum |
|--------|---------:|
| Full logo | 160 px |
| Symbol | 32 px |
| Favicon | 16 px |

Print

| Asset | Minimum |
|--------|---------:|
| Full logo | 30 mm |
| Symbol | 9 mm |

---

# Incorrect Usage

Do not:

- stretch the logo
- rotate the logo
- change proportions
- recolor the logo
- add shadows
- add outlines
- distort the symbol
- replace the typography
- use low-resolution versions

# Icons

Icons should be:

- outline style
- simple
- geometric
- consistent stroke width

Avoid decorative icons.

Preferred libraries:

- Lucide
- Heroicons

---

# Images

Use authentic imagery.

Preferred subjects:

- researchers
- collaboration
- manuscripts
- libraries
- archives
- publishing
- academic environments

Avoid:

- generic stock photography
- overly staged business imagery
- excessive visual effects

---

# Website

The website should maintain a clean layout.

Characteristics:

- generous whitespace
- readable typography
- subtle animations
- accessible contrast
- responsive design

---

## Buttons

Primary

- green background
- white text

Secondary

- transparent
- outlined

---

## Cards

Cards should have:

- rounded corners
- subtle borders
- consistent spacing

Avoid heavy shadows.

---

# Documentation

Documentation uses:

- Docusaurus
- Markdown
- Mermaid diagrams
- syntax highlighting

The documentation should prioritize clarity over decoration.

---

# Accessibility

Always ensure:

- sufficient contrast
- scalable typography
- keyboard accessibility
- descriptive alternative text
- accessible PDFs

Accessibility is a core value of OMI.

---

# Brand Voice

Write in a style that is:

- precise
- professional
- welcoming
- technically accurate
- internationally understandable

Avoid:

- marketing buzzwords
- exaggerated claims
- unnecessary complexity
